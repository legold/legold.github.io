Javascript模块化编程(一):模块的写法

发表于 2016-09-05   |  

javascript模块化编程对于对于日渐复杂的前端网页来说很有必要，因此掌握js模块化对于前端程序员来说也是必须学习与掌握的。

原始写法

模块就是实现特定功能的一组方法。
只要把不同的函数（以及记录状态的变量）简单地放在一起，就算是一个模块。

上面的函数m1()和m2()，组成一个模块。使用的时候，直接调用就行了。
但是这种做法的缺点很明显：”污染”了全局变量，无法保证不与其他模块发生变量名冲突，而且模块成员之间看不出直接的关系。当团队配合的时候，多人来写Js,就会很容易出现函数名或者变量名冲突，然后开启嘴炮模式，不断扯皮。

对象写法

为了解决上面的缺点，可以把模块写成一个对象，所有的模块成员都放在这个对象里面。

上面的函数m1()和m2(),都封装在module1对象里。使用的时候，就是调用这个对象的属性。

module1.m1();

但是，这样的写法会暴露所有模块成员，内部状态可以被外部改写。比如，外部代码可以直接改变内部计数器的值。

module1._count = 5;

立即执行函数写法(IIFE)

使用”立即执行匿名函数“（Immediately-Invoked Function Expression，IIFE），也有叫做”自执行匿名函数”，可以达到不暴露私有成员的目的。

使用上面的写法，外部代码无法读取_count变量。

console.log(module1._count); //undefined

宽放大模式（Loose augmentation）

在浏览器环境中，模块的各个部分通常都是从网上获取的，有时无法知道哪个部分会先加载。如果采用上一节的写法，第一个执行的部分有可能加载一个不存在空对象，这时就要采用”宽放大模式”。

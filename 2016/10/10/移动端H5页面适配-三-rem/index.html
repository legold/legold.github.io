移动端H5页面适配(三)rem

发表于 2016-10-10   |  

rem，让移动开发更为简单。



众所周知，在rem出现之前，我们都是用百分比配合px，再利用媒体查询写响应式，如今有了rem，我们可以在写之前就设置好基准值，然后按照设计稿来写就OK了。

何为rem

rem(font size of the root element)是指相对于根元素的字体大小的单位。简单的说它就是一个相对单位。看到rem大家一定会想到em（foot size of the element）是指相对于父元素的字体大小的单位。它们之间其实很相似，只不过一个计算的规则是依赖根元素一个是依赖父元素计算。

font-size:62.5% 或者 625%

从遥远的IE6到版本帝Chrome他们都约定好了，根元素默认的font-size都是16px。rem的兼容性如何，经过caniuse的测试，IE9+，Firefox、Chrome、Safari、Opera 的主流版本都支持了，我可以放肆的使用 rem 了。
可是，W3C 那些家伙为什么要闲着下面疼的新增这样一个单位呢？它和 em 有啥区别呢？
我们知道 em 的计算是基于父级元素的，在实际使用中给我们的计算带来了很大的不便。所以 rem 的出现解救了我这样不会算术的人，再也不用担心父级元素的 font-size 了，因为它始终是基于根元素（html） 的。
比如默认的 html font-size=16px，那么我想设置12px 的文字就是：12÷16=0.75（rem）
当然，你可以引入 CSS 预处理工具（Sass、LESS 、Stylus等）自动计算 rem 值，这里就不一一举例了。
但是像我这样的懒人或者团队开发中还没有引入 CSS 预处理工具的该肿么办呢？只能搬个计算器啪啪啪了吗？别急，你还可以变通一下。我们改变一下 html 的默认 font-size=10px 不就好计算了嘛！Like this：

注：在Chrome下，默认最下字体为12px,可以设置font-size: 625%，其他以此类推。
经过测试，PC上最小渲染px为12px，低于12px的都会被渲染为12px。
因此如果设置 62.5%后就有 1rem = 10px,但是设置成这样会有一个bug,
就是如果一个元素设置了font-size为1rem,我们预想着渲染成10px，但是由于网页最小是12px，所以这时会被强制渲染成12px，这样实际上就是 1rem = 12px,这和我们预想的就不一样了。
rem设置的基准值不能小于页面默认的最小渲染px（12px）。
所以可以考虑设置为625%。然后1rem = 100px。这样既方便计算，又不会出现实际渲染与预想的出现偏差。

需要注意的是，为了兼容不支持 rem 的浏览器，我们需要在 rem 前面写上对应的 px 值，这样不支持的浏览器可以优雅降级。其实不用太纠结是默认的 font-size:100%，还是设置为 font-size:62.5%，如果你引入 了 CSS 预处理工具那么自然可以使用默认值，如果由于其他原因使用 font-size:62.5% 也无可厚非，完全可以在 body 中重置回你需要的默认 font-size。

Gulp项目构建总结

发表于 2016-11-01   |  

一个优秀的前端工程师，在处理一个规模或大型的项目时，必定会用到一些项目自动化构建工具。比如Grunt、Gulp、FIS3、webpack等。这次我来总结一下Gulp的用法。
使用 Gulp 来进行文件压缩，文件合并，js语法检查，css自动添加浏览器前缀，自动刷新页面，文件重命名，less/sass自动编译等。配置好 gulpfile.js文件，一个命令让整个流程都跑起来，想想也是非常的酷炫。

Gulp是基于流的管道式自动化构建系统。也是基于NodeJs的自动任务运行器， 她能自动化地完成 javascript/coffee/sass/less/html/image/css 等文件的的测试、检查、合并、压缩、格式化、浏览器自动刷新、部署文件生成，并监听文件在改动后重复指定的这些步骤。

准备工作

  • 安装NodeJS应用程序（现在nodejs都继承了npm包，所以不用单独安装）
  • 全局安装Gulp
  • 本地安装（作为项目的开发依赖devDependencies）

安装nodeJS

到nodejs官网下载一个nodejs安装程序，然后安装nodejs，点击默认安装就行了。此时，会自动给系统添加一个环境变量，因此在系统的Path环境变量中就会有nodejs的安装路径，如果没有，自己手动添加即可。
由于现在nodejs都集成了npm，因此，打开命令行工具后，依次输入 node -v , npm -v 看看是否能够成功打印出版本号，如果觉得npm版本过低，可以 用 npm i npm -g更新。i是install的缩写。此外还可以用 npm config get prefix 找到npm的安装路径。

全局安装Gulp

安装完成后，gulp就安装在了npm下面的node_modules文件夹下。

然后增加系统NODE_PATH的环境变量，一般都是在 C:\Users\Administrator(这个根据你你自己的定)\AppData\Roaming\npm\node_modules
设置完成后，cmd中输入 gulp -v ，如果能够正确输出版本号，说明下载gulp成功，环境配置也成功了。

本地安装项目依赖（devDependencies）

本地安装gulp项目，作为开发依赖。

实战

此时我们找个位置，比如桌面上，新建一个文件夹，建好目录结构。比如取名 gulp-demo，完后我们cd desktop/gulp-demo,就进入了gulp-demo项目了。

首先，我们输入命令npm init,一路enter,该命令会保存我们的所用到的以及配置的信息，保存在package.json。方便我们在另一个环境上按照上面的信息来安装包。然后我们需要在本地，也就是在项目内安装gulp以及各种模块。 npm i gulp –save-dev （–save就会将模块信息保存到package.json中） 。

注： 使用 npm init 创建一个 package.json 存储依赖关系等配置信息
然后我们安装本地开发依赖

再在根目录新建一个gulpfile.js文件，来配置我们的任务。

安装依赖插件：

上述命名依次安装了 gulp 、 gulp-sass 、 gulp-autoprefixer 、 browser-sync 、 gulp-notify 等常用插件，其中：
gulp-sass 用于将 Sass 文件编译为 CSS 文件
gulp-autoprefixer 根据浏览器版本自动处理添加浏览器前缀
browser-sync能让浏览器实时、快速响应文件更改（html、js、css、sass、less等）并自动刷新页面
gulp-notify 用于任务提醒

default任务

gulp会默认执行该任务，是程序运行时的默认切入点，也是在项目根目录运行gulp时执行的任务。

子任务

通过在默认任务的第二个参数中指定其他任务即可，程序会依次执行 myTaskOne、myTaskTwo、default，我们可以对他们进行任务排序

最后这个运行gulp时会先依次执行 myTaskOne和 myTaskTwo任务。然后再执行 default任务，这个任务执行的语句就是function函数里面的语句。
